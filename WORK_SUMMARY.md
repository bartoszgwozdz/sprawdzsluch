# PDF Report — UX/UI Review & Redesign (implemented)

Premium redesign of the paid hearing-test report generated by `backend-pdf`
(Handlebars + Puppeteer). Goal: a document that looks like it came from an
audiology clinic and visibly justifies the 24,99 zł price — while matching the
SprawdźSłuch frontend brand.

Sample output: [`test-report.pdf`](test-report.pdf) — regenerate with
`node backend-pdf/scripts/gen-sample.js`.

---

## Review method
- Captured the live frontend (`/` and `/results/`) on `localhost:3000` to lock the
  real design tokens.
- Generated the **previous** template as a baseline and rendered every page to
  image for side-by-side assessment.
- Ran the `ui-ux-pro-max` design-system pass for healthcare documents
  (accessibility-first, no neon, no AI gradients, WCAG contrast).

## What was wrong with the old report
| # | Issue | Impact |
|---|-------|--------|
| 1 | **Off-brand** — serif *Playfair Display* logo + `#0066CC` blue | Looked like a different company than the site the customer just used (`#004AAD`, Montserrat) |
| 2 | **Clinically wrong audiogram** — 0 dB at the bottom, curve rising as hearing worsens | Backwards from every real audiogram (0 dB = top). Undermines credibility |
| 3 | No severity zones, no reference values, no legend | Customer can't tell if a number is good or bad |
| 4 | **Padded pagination** — 4 pages, each ~50% empty | Feels thin / not worth paying for |
| 5 | Verdict buried — result on page 4, after the table | The one thing the customer wants is last |
| 6 | No actionable next steps, no page numbers, no report identity | Doesn't read like a medical document |

## What the redesign does
**Brand match (1:1 with the frontend)**
- Single family **Montserrat** (400–800); dropped the serif.
- Logo rebuilt exactly like the site: `Sprawdź`(blue) `Słuch`(green) `.pl`.
- Palette: primary `#004AAD`, accent green `#00C853`, light `#E6F0FF`, bg `#F8F9FA`.

**Clinical-grade structure & balanced 3-page layout**
- **Page 1 — identity → verdict → proof:** tight **letterhead** (report no. +
  issue date), a compact **Dane badania** strip (patient, ID, date, paid-chip),
  the **result hero** (plain-language verdict + a green→red severity meter at the
  patient's PTA + key stats), and the **audiogram** — all on one full page.
- **Audiogram, rebuilt**: 0 dB at top (correct), shaded BIAP severity zones
  (Norma / Lekki / Umiarkowany / Znaczny / Głęboki), per-point dB labels, a
  **logarithmic X-axis from 250 Hz to the patient's max audible frequency** with a
  dashed green “Maks. zakres słyszalny” marker (mirrors the frontend's range chart).
- **Page 2 — details:** a **“Zakres słyszanych częstotliwości”** card with a
  **horizontal spectrum bar** (green log-scale fill from 3→20 kHz up to the max
  audible frequency, with a “Twój zakres” marker — mirroring the preview image and
  the frontend results chart) plus a rating verdict ported from the frontend's
  tiers (Bardzo dobry ≥17 kHz / Dobry ≥14 / W normie ≥10 / Lekkie ograniczenie ≥6 /
  Znaczące ograniczenie <6, colour-coded badge), the **results table**
  (per-row severity bars + badges + legend), and the **interpretation + next
  steps** (green checklist, adapts to the result).
- **Page 3 — references:** “Jak czytać wynik” severity table (patient's own band
  highlighted), a “Co oznaczają badane częstotliwości” everyday-sounds guide,
  the **disclaimer**, and a **methodology/signature** block.
- Branded footer with `Strona X z Y` on every page.

**Medical accuracy (audited & unified)**
- Found **3 conflicting hearing-loss thresholds** across files (dataService used
  20/40/**60/80**; pdfService, audiogram zones and the reference table used BIAP
  20/40/**70/90**). A 65 dB row showed "Znaczny ubytek" text with a *moderate*
  colour while the guide called 65 dB "Umiarkowany" — a contradiction.
- **Standardised everything on BIAP (20/40/70/90)** — the European/Polish clinical
  standard — so per-frequency text, badge colour, hero status/colour, audiogram
  zones and the reference table all agree. Verified at every boundary.
- Overall result now uses a proper **PTA** (average of speech frequencies
  500/1000/2000/4000 Hz) instead of a flat mean of all tones.
- Dropped the "dB **HL**" wording (the test produces uncalibrated web-audio gains,
  not calibrated Hearing Level) — now plain "dB", matching the screening disclaimer.

**Accessibility**
- All status colors meet WCAG AA on their backgrounds; severity is conveyed by
  text + position, never color alone. Icons are inline SVG (no emoji).

## Result
- **4 sparse pages → 3 dense, balanced pages** (page 1 and 2 full; page 3 carries
  the references and closing notes).
- Tightened the letterhead and the header→hero spacing; eliminated the page-1
  bottom gap by pairing the verdict with the audiogram.
- Data-driven: verified on a *normal* result (green hero, marker far-left, receipt
  hidden) and a *mixed-loss* result (amber hero, correct band highlighted).
